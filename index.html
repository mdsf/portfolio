Substitution Cipher

The project is essentially an encryption/decryption program that can be used to send and recieve files and information in a secure manner. The program is made in Java and has a basic GUI functionality.

Project 1

Compiling and execution from command line:

To compile the source code from the command line, use the following command. After which classes will be generated in the desired folder. (test1 is the package in this case)

Javac test1\GUI.java test1\CryptographicUtilities.java test1\Encryption.java test1\OTP.java test1\Substitution.java

After compilation, to execute and run the program use the following command.

Java -classpath C:\Users\UserName\Desktop\project_test_compile test1.GUI outputGui

Interface:

The GUI contains 7 buttons that are all functional except Cryptanalysis and each of them contains a unique operation.

1.      Cipher key(s).

2.      OTP key(s).

3.      Substitution Encrypt/Decrypt.

4.      One time pad Encrypt/Decrypt.

5.      MD5 Checksum.

6.      File Compare.

7.      Cryptanalysis.  (When clicked displays that functionality was not implemented)

 

Encrypt or Decrypt a File with Substitution:

In order to encrypt or decrypt a file with Substitution method we have to generate the encryption and decryption keys first.

1.       Click on the Cipher key(s) button. The program will ask you to select a location for encrypt key, choose your folder and store it in a new text file. Same procedure applies for decrypt key as well.

2.       After successfully generating and storing your keys. Click on the Substitution Encrypt/Decrypt button. The program will ask you to select a file containing your key. For encryption select the file containing the encrypt key and for decryption the same procedure follows on.  Then the program will ask you to select a file that you need to encrypt or decrypt. Do that and lastly it will ask you to select a file to store your result.

Encrypt or Decrypt a File with One time Pad:

In order to encrypt or decrypt a file with One Time Pad method we have to generate the encryption and decryption keys first. In this case the key for encryption and decryption are the same.

1.       Click on the OTP key(s) button. The program will ask you to enter a positive key size and will check for correct input if a string or negative input is given. The key size will be rounded off to the next byte size. For e.g. if you enter 1025 key size will be 2048. The program will ask you to select a location for encrypt key, choose your folder and store it in a new text file.

 

2.       After successfully generating and storing your keys. Click on the one time pad Encrypt/Decrypt button. The program will ask you to select a file containing your key.  Then the program will ask you to select a file that you need to encrypt or decrypt. Do that and lastly it will ask you to select a file to store your result.

Generating a checksum for a file:

To generate a checksum for a file click on the MD5 Checksum button.  The program will ask you to select a file that you need to generate a checksum for. Select a file containing text and then the program will ask you to save the checksum result in another file. You can only generate a checksum for one file at a time. To generate multiple checksums simply repeat the process. The source code used for generating the checksums is taken from the following website.

http://howtodoinjava.com/core-java/io/how-to-generate-sha-or-md5-file-checksum-hash-in-java/

File comparison of two files:

To compare two files click on theFile Compare button. The program will ask you to select a file that already contains a checksum generated of a file. Then the program will ask you to select a second file that contains a checksum generated for that file as well. Then the program will display whether the checksum of two files were the same or different.

Note:

 To make things easier to understand, what I did was I had two files one named sampletest.txt containing my resume and other file named sapmletest2.txt containing my English essay. First I generated the checksums for these files one by one and then I compared the checksums of these two files by the file compare button.
